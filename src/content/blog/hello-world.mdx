Every developer eventually builds a personal site. This is mine.

I've been meaning to do this for years. The usual excuses applied — not enough time, not sure what to write about, the design never felt right. At some point you just have to ship something and iterate.

## What this is

This is a place for me to think out loud about QA automation, tooling, and software quality. Some posts will be technical. Some will be more reflective — lessons from projects, things I'd do differently, opinions I've formed over 7+ years in the industry.

I'm not trying to build an audience. I'm writing to clarify my own thinking, and if something turns out to be useful to someone else, that's a bonus.

## What's coming

A few topics I want to get into:

- Playwright vs Selenium — an honest comparison after using both in production
- Why most test suites have a flakiness problem that isn't really a test problem
- Building a test framework from scratch: the decisions that matter
- CI/CD for QA teams: what actually works

That's enough for now. Let's see how this goes.
